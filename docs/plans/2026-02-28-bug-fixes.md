## T1：修复 clear() 时间方向

### 问题
`Instant.now().plusMillis(extra.toMillis())` 导致删除"未来才过期"的锁。

### 修复
`GeneralJdbcOperator.java:104`
```java
// before
Instant time = Instant.now().plusMillis(extra.toMillis());
// after
Instant time = Instant.now().minusMillis(extra.toMillis());
```

## T2：修复 release 竞态（token 方案）

### 涉及文件
1. `idempotent-guard-jdbc/.../HoldState.java` — 增加 `holdToken` 字段
2. `idempotent-guard-jdbc/.../JdbcOperator.java` — `delete()` 签名加 token 参数
3. `idempotent-guard-jdbc/.../GeneralJdbcOperator.java` — INSERT/DELETE SQL 更新
4. `idempotent-guard-jdbc/.../GeneralJdbcGuard.java` — release 传 token
5. `db/schema-mysql.sql` / `db/schema-postgre.sql` — 加 hold_token 列

### 修复细节

**HoldState** 增加字段：
```java
private final String holdToken;
```

**JdbcOperator** 接口：
```java
boolean delete(String id, String token);
```

**GeneralJdbcOperator** SQL：
```sql
-- INSERT 加 hold_token
INSERT INTO %s (id,start_time_utc,expire_time_utc,hold_by,hold_hint,hold_token) VALUES (?,?,?,?,?,?)

-- DELETE 加 token 校验
DELETE FROM %s WHERE id = ? AND hold_token = ?
```

**GeneralJdbcGuard.release()**：
```java
public void release(Holder holder) {
    HoldState state = (HoldState) holder;
    jdbcOperator.delete(state.getKey(), state.getHoldToken());
}
```

**schema** 新增列：
```sql
hold_token VARCHAR(36) NOT NULL DEFAULT ''
```

## T3：修复 ClearJob 单次调度

### 问题
`schedule(...)` 只执行一次。

### 修复
重构 ClearJob，将 `ScheduledExecutorService` 改为可注入（便于测试），调度改为 `scheduleAtFixedRate`：

```java
// before
this.executorService.schedule(() -> clear(...), executeDelay, TimeUnit.SECONDS);
// after
this.executorService.scheduleAtFixedRate(() -> clear(...), executeDelay, executeDelay, TimeUnit.SECONDS);
```

构造函数增加 `ScheduledExecutorService` 参数（默认构造保持兼容）。
